Аннотация

В данном программном документе приведено описание программного средства «Модуль управления корпоративной беспроводной инфраструктурой». Программное средство представляет собой Web-приложение, реализующее интерфейс управления корпоративной беспроводной инфраструктурой. 
Администратору сети предоставляются возможности по управлению доменами, привязанными к ним вещаниями SSID, физическими точками доступа (включая работу со списками точек доступа, ожидающих добавления), клиентами и их группами. В каждом случае (за исключением беспроводных клиентов) управление подразумевает возможность добавления, удаления, изменения и просмотра списка соответствующих объектов администрирования. Управление беспроводными клиентами осуществляется преимущественно через RADIUS-сервер, модуль лишь поддерживает функции просмотра их списка и изменения дополнительных параметров.
Также администратору предоставляются возможности по мониторингу сети: просмотр карт помещений, в которых развернута корпоративной беспроводная инфраструктура, изучение параметров и статистики виртуальных точек доступа, закрепленных за беспроводными клиентами, просмотр сведений о беспроводных клиентах. Программная реализация данных функций в значительной степени упрощает выполнение рутинных операций, сокращает время развертывания и перенастройки сети.
Модуль управления корпоративной беспроводной инфраструктурой написан на языке программирования PHP5. В процессе своей работы для реализации заложенных функций он с помощью протокола HTTP взаимодействует с модулем расширения сетевой операционной системы (с помощью открытого высокоуровневого API), а также с модулем визуализации работы корпоративной беспроводной инфраструктуры. Разработанный модуль управления поддерживает работу с физическими точками доступа фирмы Juniper (взаимодействие происходит с контроллером RingMaster), а также с точками доступа других фирм, включая TP-Link.
Оформление программного документа «Описание программы» произведено по требованиям ЕСПД  (ГОСТ 19.101-77, ГОСТ 19.103-77, ГОСТ 19.104-78, ГОСТ 19.105-78, ГОСТ 19.106-78, ГОСТ 19.402-78, ГОСТ 19.604-78).

Непрерывная работа модуля управления корпоративной беспроводной инфраструктурой обеспечивается Web-сервером Apache, который передает управление модулю при поступлении соответствующих запросов от браузера администратора.
При запуске модуль управления корпоративной беспроводной инфраструктурой считывает конфигурационные данные и инициализирует внутренние структуры данных. Затем получает тип и параметры HTTP-запроса, в зависимости от типа запроса выполняет соответствующие действия. В процессе их выполнения он взаимодействует с модулем расширения сетевой операционной системы (при управлении беспроводными точками доступа TP-Link, мониторинге виртуальных точек доступа), модулем визуализации корпоративной беспроводной инфраструктуры (при работе с картами здания), контроллером RingMaster беспроводных точек доступа Juniper (при работе с точками доступа фирмы Juniper), RADIUS-сервером (для получения сведений о зарегистрированных клиентах).

5 Вызов и загрузка

Модуль запускается в качестве Web-приложения сервера Apache. В конфигурационный файл “httpd-vhosts.conf” хостов Apache должны быть добавлены следующие строки:
<VirtualHost *:80>
    ServerAdmin postmaster@<имя_сайта>
    DocumentRoot "<путь_к_корневому_каталогу>"
    ServerName <имя_сайта>
    ServerAlias www.<имя_сайта>
    ErrorLog "logs/<имя_сайта>-error.log"
    CustomLog "logs/<имя_сайта>-access.log" combined
</VirtualHost>
Здесь <имя_сайта> - доменное имя сайта, например, “site.com”, <путь_к_корневому_каталогу> - путь к корневому каталогу сайта.
Доменное имя сайта также должно быть указано в файле “hosts”, оно также может быть прописано в базе данных DNS-сервера.
Для запуска сервера Apache необходимо выполнить команду “sudo apachectl start”, для остановки – “sudo apachectl stop”.
 
6 Входные данные

Входными данными для программы является: 
а) информация из файла настроек. Характер данных – текстовая информация, организация данных – построчный текст, каждая строка несет отдельную смысловую нагрузку. Данные требуют предварительной подготовки с целью указания точного адреса контроллера, шлюза, адреса сетевой карты. Формат данных определяется форматом UNIX файлов настроек в виде «<#> параметр = значение1<, значение2, …>», где элементы в угловых скобках являются необязательными, «параметр» - это название определяемого вновь или переопределяемого параметра, «значение1», «значение2» - произвольные сочетания латинских букв, цифр, знаков препинания (кроме запятой), знак решетки в начале строки означает комментарий, не принимаемый программой во внимание. Кодировка данных – «UTF-8»;
б) сведения из базы данных. Характер данных – структурированная информация, организация данных – двоичная структура, реализованная объектно-реляционным отображением модели базы данных. Предварительная подготовка требуется при чтении и поиске строк объектов, осуществляется автоматически при  обращении к атрибутам объектов классом объектно-реляционной модели базы данных. Формат данных – классы объектов с атрибутами нестрогой типизации, например «$model->attributes», где «$model» - имя объекта, «attributes» - имя атрибута объекта. Кодировка данных – «UTF-8»;

7 Выходные данные

Выходным данными являются:
а) сведения, сохраняемые в базу данных. Характер данных – структурированная информация, организация данных – двоичная структура, реализованная объектно-реляционным отображением модели базы данных. Предварительная подготовка требуется при чтении и поиске строк объектов, осуществляется автоматически при  обращении к атрибутам объектов классом объектно-реляционной модели базы данных. Формат данных – классы объектов с атрибутами нестрогой типизации. Кодировка данных – «UTF-8»;
б) генерируемые Web-страницы. Характер данных – текстовая информация, организация данных – текст с разделителями HTML, каждый блок текста, ограниченный разделителями может содержать вложенный блок подобной структуры. Каждая страница предварительно генерируется из набора объектов и структур модуля интерпретатором языка PHP. Формат данных – HTML.  Кодировка данных – «UTF-8»;
в) генерируемые и изображения. формата PNG. Характер данных – двоичная информация, организация данных – по структуре PNG. Предварительная подготовка осуществляется на стороне сервера, где генерируется изображение и затем преобразуется в формат PNG. Формат данных – PNG. Кодировка данных – двоичная информация, организованная по восемь бит.
